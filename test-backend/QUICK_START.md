# 🚀 本地测试服务器快速启动指南

## 问题诊断

你的应用显示以下错误：
```
failed to connect to /10.0.2.2 (port 8080)
```

这意味着**本地 API 测试服务器没有在运行**。

---

## ⚡ 3 步快速启动

### Step 1️⃣: 打开命令行
Windows (PowerShell 或 CMD):
```powershell
cd E:\Work\AIpet\test-backend
```

### Step 2️⃣: 启动服务器

**Windows:**
```powershell
.\run_server.bat
```

或在文件浏览器中**双击** `run_server.bat`

**Mac/Linux:**
```bash
./run_server.sh
```

### Step 3️⃣: 验证服务器已启动

你应该看到：
```
==================================================
🚀 Local AI Pet API Server 启动中...
==================================================
📍 服务器运行在: http://0.0.0.0:8080
📱 模拟器访问地址: http://10.0.2.2:8080
💡 API 端点: /api/chat, /api/health, /
⌨️  按 Ctrl+C 停止服务器
==================================================
```

## ⚠️ 常见问题

### 问题 1: "Python 未找到"
**解决**: 安装 Python 3.8+
- 下载: https://www.python.org/downloads/
- 安装时**勾选** "Add Python to PATH"

### 问题 2: "模块 Flask 未找到"
**解决**: 手动安装依赖
```powershell
pip install Flask Flask-CORS
```

### 问题 3: "端口 8080 已占用"
**解决**: 修改 `local_api_server.py` 最后一行，改端口号
```python
app.run(host='0.0.0.0', port=9000)  # 改成 9000 或其他
```
然后在应用设置中也改成相同的端口

### 问题 4: "应用仍然无法连接"
**检查清单**:
- [ ] 服务器确实在运行（命令行窗口仍打开）
- [ ] 防火墙未阻止端口 8080
- [ ] 应用配置中使用了正确的地址 `http://10.0.2.2:8080`
- [ ] 模拟器已启动
