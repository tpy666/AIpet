## 项目概述

**AIpet** 是一个 Android 宠物聊天应用，集成了多个 AI API（本地、OpenAI、豆包），支持真实的宠物个性化对话。

### 核心特性
- ✅ **多 API 支持**：本地测试服务器、OpenAI、字节跳动豆包
- ✅ **宠物设定系统**：物种、性格、说话风格个性化配置
- ✅ **系统提示词**：根据宠物特征自动生成 AI 指导词
- ✅ **实时日志查看**：聊天日志和错误日志完整记录
- ✅ **灵活配置界面**：API 配置、连接测试、日志查看一站式管理
- ✅ **自动降级机制**：API 失败自动使用本地模拟
- ✅ **网络安全配置**：支持 HTTP 本地开发调试

### 目标用户
- 📱 移动应用开发者
- 🤖 AI 聊天应用开发者
- 🐕 宠物爱好者

## 核心功能详解

### 1. 宠物提示词系统

#### PetPromptGenerator
根据宠物信息自动生成系统提示语：

```java
String prompt = PetPromptGenerator.generatePrompt(petInfo);
// 输出示例：
// "你是一只名叫糖丸的温柔长毛猫。
//  性格特点：温柔、友善
//  说话风格：卖萌、可爱
//  用表情符号增强表现力（如"喵~"）"
```

**支持的属性**：
- 宠物名字
- 物种（猫、狗、兔子等）
- 性格（温柔、调皮、活泼、高冷等）
- 说话风格（卖萌、沙哑、可爱等）
- 外观描述

### 2. 多 API 支持

#### 本地测试服务器
- **路由**：`POST /api/chat`
- **端口**：8080
- **特点**：无需网络，完全免费
- **回复**：根据宠物性格生成多样化回复

#### OpenAI API
- **基础 URL**：`https://api.openai.com/v1`
- **端点**：`/chat/completions`
- **特点**：高质量回复，支持 gpt-3.5-turbo/gpt-4
- **需要**：API Key

#### 豆包 API
- **基础 URL**：`https://ark.cn-beijing.volces.com/api/v3`
- **端点**：`/responses`
- **特点**：多模态支持（文本+图片），国产
- **需要**：Bearer Token

### 3. 自动重试机制

```
请求失败 (可恢复错误)
    ↓
等待延迟 (1秒)
    ↓
重新发送 (最多3次)
    ↓
若全部失败 → 自动降级到本地模拟
```

**关键参数**：
- 最大重试次数：3
- 初始延迟：1 秒
- 延迟倍数：2（指数退避）
- 最大延迟：8 秒

### 4. 日志系统

#### 记录的信息
```
用户消息:  "用户: 你好 [Pet: 糖丸]"
API请求:  "Method: POST, URL: http://10.0.2.2:8080/api/chat"
API响应:  "Status: 200, Response: 喵~你好呀！"
宠物回复:  "[糖丸]: 我好像不太感兴趣呢..."
错误信息:  "API_ERROR: NETWORK_ERROR - [详细错误信息]"
降级:     "FALLBACK: Using fallback mechanism: API 请求失败"
```

#### 查看日志
- 应用内：设置 → 查看聊天日志/错误日志
- 文件系统：`/sdcard/Android/data/com.example.aipet/cache/chat_logs/`
- logcat：`adb logcat | grep "APIpet"`

## 已知问题和解决方案

### 问题 1：HTTP 404 错误（已解决 ✅）

**症状**：请求失败，返回 404

**原因**：
- 本地服务器路由为 `/api/chat`
- 但应用之前直接发送到基础 URL
- 导致端点不匹配

**解决方案**：
- ✅ testConnection() 现在根据提供商智能追加端点
- ✅ 本地后端追加 `/api/chat`
- ✅ OpenAI 追加 `/v1/chat/completions`
- ✅ 豆包追加 `/responses`

### 问题 2：连接测试虚假响应（已解决 ✅）

**症状**：连接测试总是显示成功，但实际 API 调用失败

**原因**：
- testConnection() 之前只是延迟 1.5 秒后显示成功
- 没有真实的 HTTP 请求

**解决方案**：
- ✅ testConnection() 现在发送真实的 POST 请求
- ✅ 显示真实的 HTTP 状态码
- ✅ 记录详细的错误信息

### 问题 3：API 超时（已解决 ✅）

**症状**：请求超时（30秒）

**原因**：
- 网络连接不稳定
- 本地服务器没有运行

**解决方案**：
- ✅ 超时时间调整为 60 秒
- ✅ 本地服务器超时调整为 90 秒
- ✅ 自动重试 3 次

### 问题 4：宠物回复模板化（已解决 ✅）

**症状**：无论问什么，宠物回复都是固定模板

**原因**：
- 本地 API 服务器的 `get_pet_response()` 函数过于简单
- 没有根据宠物性格生成多样化回复

**解决方案**：
- ✅ 增强了 `get_pet_response()` 函数
- ✅ 分析宠物性格特征
- ✅ 生成对应的多样化回复
- ✅ 支持宠物类型拟声词

### 问题 5：本地服务器 Python 依赖（已解决 ✅）

**症状**：No module named 'flask'

**解决方案**：
```powershell
# 使用 Python 启动器安装依赖
py -m pip install flask flask-cors

# 启动服务器
py local_api_server.py

# 或简单运行脚本
.\run_server.bat
```

