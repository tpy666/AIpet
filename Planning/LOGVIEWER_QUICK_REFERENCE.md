# 📋 日志查看器 - 快速参考卡

## 🎯 一句话说明
**LogViewerActivity** 是应用的日志查看和调试工具，可以实时监控应用运行情况。

---

## 🚀 3 秒快速开始

### 步骤 1️⃣ - 打开
```
主菜单 → 点击 "📋 查看日志"
```

### 步骤 2️⃣ - 查看
```
日志会自动显示，新日志出现在底部
```

### 步骤 3️⃣ - 操作
```
• 点击查看详情
• 菜单过滤日志
• 搜索框搜索内容
```

---

## 🎨 主要功能

| 功能 | 操作 | 快捷键 |
|------|------|--------|
| **查看日志** | 点击列表项 | - |
| **过滤日志** | 点击菜单选择等级 | - |
| **搜索日志** | 在搜索框输入 | - |
| **查看详情** | 点击日志项 | - |
| **复制日志** | 点击详情的"复制"按钮 | - |
| **导出日志** | 菜单 → 📤 导出 | - |
| **清空日志** | 菜单 → 🗑️ 清空 | - |
| **刷新日志** | 菜单 → 🔄 刷新 | - |

---

## 📊 日志等级对应

```
🔵 DEBUG    → 开发调试信息，最详细
🟢 INFO     → 常规运行信息
🟡 WARNING  → 警告信息，可能有问题
🔴 ERROR    → 错误信息，需要立即处理
```

---

## 💡 常用操作

### 查找 API 错误
```
1. 菜单 → 🔴 ERROR
2. 搜索框输入 "Api" 或 "Network"
3. 查看所有 API 错误
```

### 追踪用户操作
```
1. 菜单 → 🟢 INFO
2. 搜索框输入 "ChatActivity"
3. 查看用户操作序列
```

### 性能分析
```
1. 菜单 → 🟡 WARNING
2. 查找 "timeout" 或 "slow"
3. 识别性能瓶颈
```

### 分享给开发者
```
1. 菜单 → 📤 导出
2. 选择分享方式
3. 发送给开发者
```

---

## 🔍 搜索技巧

| 搜索词 | 效果 |
|--------|------|
| `api` | 查找所有 API 相关日志 |
| `error` | 查找所有错误（不分大小写） |
| `message` | 查找特定消息内容 |
| `timeout` | 查找超时相关 |
| `ChatActivity` | 查找特定模块 |

搜索支持：
- ✅ 大小写不敏感
- ✅ 部分匹配
- ✅ 实时过滤

## ⚡ 快捷菜单

点击右上角菜单按钮可以：

```
过滤选项：
├─ 📋 全部 ..................... 显示所有日志
├─ 🔵 DEBUG .................... 仅 DEBUG
├─ 🟢 INFO ..................... INFO 以上
├─ 🟡 WARNING .................. WARNING 以上
└─ 🔴 ERROR .................... 仅 ERROR

操作选项：
├─ 🔄 刷新 ..................... 重新加载
├─ 📤 导出 ..................... 分享日志
└─ 🗑️ 清空 ..................... 清除日志
```

---

## 🆘 故障排查

| 问题 | 解决方法 |
|------|---------|
| 没有日志 | 点击"🔄 刷新"或运行应用操作 |
| 日志很多 | 菜单选择高级别过滤 |
| 找不到特定日志 | 使用搜索框搜索关键字 |
| 想清空日志 | 菜单 → "🗑️ 清空"（无法撤销） |

## 📝 日志示例

### DEBUG 级别
```
[10:23:45.123] DEBUG - ApiClient: 配置 URL: https://api.example.com

### INFO 级别
```
[10:23:46.001] INFO - ChatActivity: 用户发送消息

```

### WARNING 级别
```
[10:23:47.100] WARN - ApiClient: 连接超时，进行第1次重试
[10:23:48.200] WARN - ChatLogger: 内存日志已满，删除旧日志

### ERROR 级别
```
[10:23:49.500] ERROR - ApiClient: 网络请求失败，错误: Connection refused
[10:23:50.600] ERROR - ChatActivity: 空指针异常: userInput is null
```

## 💾 导出日志

**用途**：当需要将日志发送给开发者和技术支持

**操作**：菜单 → 📤 导出 → 选择分享方式

**内容包括**：
- 应用会话 ID（用于追踪）
- 导出时间
- 日志总数
- 完整的所有日志记录

## 🎓 在代码中使用

### 获取 Logger
```java
ChatLogger logger = ChatLogger.getInstance(context);
```

### 记录日志
```java
logger.debug("TAG", "调试信息");
logger.info("TAG", "信息");
logger.warning("TAG", "警告");
logger.error("TAG", "错误", exception);
```

### 查看日志
```java
// 启动日志查看器
startActivity(new Intent(context, LogViewerActivity.class));
```

## 📞 需要帮助？

1. **开不了日志查看器** → 确保在主菜单中有"查看日志"按钮
2. **没有看到日志** → 点击菜单的"🔄 刷新"
3. **想搜索日志** → 在顶部搜索框输入关键字
4. **想分享日志** → 点击菜单的"📤 导出"
5. **想清除日志** → 点击菜单的"🗑️ 清空"（警告：无法撤销！）
