# 📋 日志查看器 (LogViewerActivity) - 完整指南

## 🎯 功能概述

LogViewerActivity 是一个专业的日志管理和查看工具，用于实时监控和调试 AIpet 应用的运行情况。

### 核心功能
- ✅ **实时日志显示** - 查看应用运行过程中的所有日志
- ✅ **日志等级过滤** - DEBUG / INFO / WARNING / ERROR 多级过滤
- ✅ **关键字搜索** - 快速定位特定日志内容
- ✅ **日志详情查看** - 点击查看完整日志信息
- ✅ **日志导出** - 导出日志用于分享或存档
- ✅ **日志清空** - 清理历史日志记录
- ✅ **智能刷新** - 实时更新最新日志

## 🚀 快速开始

### 1. 打开日志查看器

#### 方式 A：从主菜单打开
```
1. 打开 AIpet 应用
2. 主页 → 点击 "📋 查看日志" 按钮
3. 进入日志查看器
```

#### 方式 B：从代码启动
```java
// 在 Activity 中启动日志查看器
Intent intent = new Intent(MainActivity.this, LogViewerActivity.class);
startActivity(intent);
```

#### 方式 C：使用快捷方式
```java
// 从任何 Activity 直接跳转
startActivity(new Intent(getContext(), LogViewerActivity.class));
```

### 2. 查看日志

**简单方式：**
- 日志会自动显示应用运行过程中的所有事件
- 新日志会自动出现在列表底部
- 列表会自动向下滚动显示最新日志

**点击查看详情：**
- 点击任何日志项查看完整内容
- 可以复制或分享日志内容

### 3. 过滤日志

使用菜单过滤不同等级的日志：

**菜单选项：**
- 📋 **全部** - 显示所有日志
- 🔵 **DEBUG** - 仅显示 DEBUG 级别日志（最详细）
- 🟢 **INFO** - 显示 INFO 及以上日志
- 🟡 **WARNING** - 显示 WARNING 及以上日志
- 🔴 **ERROR** - 仅显示 ERROR 级别日志（最重要）

**使用方法：**
1. 点击右上角菜单按钮
2. 选择要过滤的日志等级
3. 列表会自动更新

### 4. 搜索日志

**快速搜索：**
1. 在顶部搜索框输入关键字
2. 日志会实时过滤匹配的内容
3. 支持大小写不敏感搜索

**搜索示例：**
- 搜索 "API" → 显示所有 API 相关日志
- 搜索 "error" → 显示包含 error 的日志
- 搜索 "ChatActivity" → 显示来自 ChatActivity 的日志

## 🔧 日志等级说明

### 1. DEBUG (🔵)
- **级别**：最低
- **用途**：调试信息，开发人员使用
- **示例**：
  ```
  [10:23:45.123] DEBUG - ApiClient: 发送请求到 http://api.example.com
  [10:23:45.234] DEBUG - ChatLogger: 日志初始化完成
  ```

### 2. INFO (🟢)
- **级别**：常规
- **用途**：应用运行信息，用户行为
- **示例**：
  ```
  [10:23:46.001] INFO - ChatActivity: 用户发送消息，内容: 你好
  [10:23:46.500] INFO - ApiClient: 收到服务器响应
  ```

### 3. WARNING (🟡)
- **级别**：警告
- **用途**：潜在问题
- **示例**：
  ```
  [10:23:47.100] WARN - ApiClient: 连接超时，进行重试
  [10:23:48.200] WARN - ChatLogger: 内存日志已满，删除旧日志
  ```

### 4. ERROR (🔴)
- **级别**：最高
- **用途**：错误和异常
- **示例**：
  ```
  [10:23:49.500] ERROR - ApiClient: 网络请求失败，错误: Connection refused
  [10:23:50.600] ERROR - ChatActivity: 空指针异常...
  ```

## 💾 日志管理

### 导出日志

**用途**：分享日志用于调试或存档

**步骤：**
1. 在日志查看器菜单中点击 "📤 导出"
2. 选择分享方式（邮件、文件、IM 等）
3. 日志会被导出为纯文本格式

**导出内容示例：**
```
=== ChatLogger 日志导出 ===
会话 ID: 1234567890
导出时间: Fri Apr 04 10:30:00 GMT+08:00 2026
日志条数: 247
===========================

[10:23:45.123] DEBUG - ApiClient: 发送请求...
[10:23:46.001] INFO - ChatActivity: 用户发送消息...
... (更多日志)
```

### 清空日志

**用途**：清理历史日志，释放内存

**步骤：**
1. 在日志查看器菜单中点击 "🗑️ 清空"
2. 确认删除提示
3. 所有日志将被清空（无法撤销）

**警告**：此操作无法撤销，请谨慎使用

### 刷新日志

**用途**：刷新日志列表，显示最新内容

**步骤：**
1. 点击菜单中的 "🔄 刷新"
2. 日志列表会重新加载

## 📊 日志统计信息

在导出日志时会包含统计信息：

```
=== ChatLogger 日志导出 ===
会话 ID: 1234567890
导出时间: Fri Apr 04 10:30:00 GMT+08:00 2026
日志条数: 247
===========================
```

**说明：**
- **会话 ID**：本次应用运行的唯一标识
- **导出时间**：日志导出的时间
- **日志条数**：内存中保存的日志总数（最多 1000 条）

## ⚙️ 配置选项

### 文件日志目录

日志文件保存位置：
```
/data/data/com.example.aipet/files/chat_logs/
```

日志文件命名规则：
```
chat_YYYY-MM-DD.log

例如：
chat_2026-04-04.log
chat_2026-04-05.log
```

### 日志文件大小限制

- **内存日志数量**：最多保存 1000 条日志在内存中
- **文件日志**：无限制（建议定期清理）

## 🐛 常见问题

### Q1：为什么看不到任何日志？
**A1：**
- 检查应用是否正在运行活动（需要有运行中的操作才会有日志）
- 尝试点击 "🔄 刷新" 按钮
- 确保没有应用 高级别过滤（如仅显示 ERROR）

### Q2：日志会永久保存吗？
**A2：**
- **内存日志**：应用关闭后会丢失（保存最多 1000 条）
- **文件日志**：保存到文件中，除非手动删除或点击 "🗑️ 清空"

### Q3：如何分享日志给开发者？
**A3：**
1. 点击菜单 → "📤 导出"
2. 选择分享方式（邮件、QQ、微信等）
3. 开发者会收到完整的日志内容

### Q4：日志查看器会影响应用性能吗？
**A4：** 
- 不会显著影响性能（日志记录优化过）
- 长时间查看较大日志列表可能略微增加内存使用

### Q5：能否在后台记录日志？
**A5：** 
- 是的，日志会始终在后台记录
- 关闭日志查看器后，日志记录仍然继续

## 🔗 相关代码

### 在代码中记录日志

**使用 ChatLogger 记录日志：**

```java
// 获取 ChatLogger 实例
ChatLogger logger = ChatLogger.getInstance(context);

// 记录不同等级的日志
logger.debug("TAG", "这是调试信息");
logger.info("TAG", "这是信息日志");
logger.warning("TAG", "这是警告日志");
logger.error("TAG", "这是错误", exception);
```

**在 ApiClient 中：**
```java
chatLogger.info("ApiClient", "发送请求: " + url);
chatLogger.error("ApiClient", "请求失败", throwable);
```

**在 ChatActivity 中：**
```java
chatLogger.info("ChatActivity", "用户发送消息: " + message);
chatLogger.info("ChatActivity", "收到服务器回复");
```
